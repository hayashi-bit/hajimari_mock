開発フィロソフィー

hajimariはAIが先に話しかける設計です。ユーザーを待たせない。
返答は2〜3文以内。最後に質問を1つ。相手の言葉を否定せず、まず受け止める。
選択肢は出さない。定型挨拶は使わない。3回に1回は予想外の角度から問いかける。

技術スタック

レイヤー 技術 理由
Frontend Plain HTML / CSS / JS フレームワーク不要。シンプルさを最優先
API Vercel Serverless (TypeScript) ゼロ設定でデプロイ。コールドスタートが速い
AI Claude (claude-sonnet-4-6) 日本語の自然さ・文脈理解の精度
Streaming Server-Sent Events (SSE) 体感速度向上。打ち込んでいる感を演出
Hosting Vercel GitプッシュでCI/CD。プレビューURLが即生成
