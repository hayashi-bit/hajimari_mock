## なぜ今これか
- JSON恒久対策・publicId(ランダム公開URL) は**実装済だが本番未反映**。本番は古いビルドのまま（`/u/1` が 200）。
- 「本番が古いまま」を毎回人目で気づくのは不確実 → **スモークスクリプトで構造的に検知**する。

## 手順

### 1. 本番デプロイ（Manus）
Manus 側で最新ビルドを本番へデプロイする。完了したら 2 へ。

### 2. デプロイ後スモーク（このスクリプト）
```bash
BASE_URL=https://liverico.app tools/smoke/post_deploy_smoke.sh
```
合格条件（どちらも満たせば新ビルド反映済み）:
- 未登録 `/api/*` が **JSON で 404**（HTML を返したら旧ビルド＝`Unexpected token '<'` の元凶）
- 連番 `/u/1` が **404/410**（200 なら旧ビルドで人数露出が残存）

`exit 1`（FAIL）の間は次へ進まない。デプロイが反映されるまで 1↔2 を繰り返す。

> ⚠️ 実行ネットワークから対象ドメインへ到達できること（egress 許可リスト）。
> 到達不可だと HTTP 000/403 になり判定できない。

### 3. Gmail E2E スモーク(A)
方法A＝**本番URLで環境変数を一時的に「開発DB＋dev用OAuthクライアント」へ切替→テスト→即戻す**。
（プレビュー/サンドボックスURLは使い捨てでOAuthが成立しないため。恒久は方法B＝独立ステージング）

絶対ルール:
- **無人時間に放置しない**（切替中は本番が開発DBを向く）。
- **テスト後すぐ本番設定へ戻す**（戻し忘れ厳禁）。チェックリストで戻しを確認。

E2E チェック項目（林本人のGmail＝チューニング母艦）:
- [ ] Gmail連携（OAuth）が 403 なく通る
- [ ] 取得プレビュー：公演名・日付・会場・枚数・ステータスが出る
- [ ] 既知の網漏れ2通（MAISONdes/eplus.co.jp・ずとまよ/pia.co.jp）が**取り込まれる**
- [ ] 宣伝（おまかせエントリー/一般発売/プレリザーブ等）が**混入しない**（LLM一次判定で除去）
- [ ] 増分同期：再実行で重複せず新着だけ増える
- [ ] 本文非保存：`gmail_analysis_cache` に本文が残らない（メタデータのみ）
- [ ] **戻し確認**：環境変数を本番DB＋本番OAuthへ戻した

抽出精度の回帰はサンドボックス側で:
```bash
export ANTHROPIC_API_KEY=sk-ant-...
python tools/gmail_eval/run_eval.py
```

スモーク全合格 ＋ Gmail E2E の上記項目クリア ＋ 環境変数を本番へ戻し済み。
