LiveLog — ライブ参加記録・管理アプリ

ユーザーが参加したライブ・コンサートの記録・管理・共有を簡単にするアプリ。e+連携、AI解析、セットリスト記録、友達機能などを提供。

公開URL：https://livelogapp-nhdbwh2e.manus.space/

LP（紹介ページ）：サービス紹介ランディングページを開く →

ブランド・カラートーン

アプリのカラートーンは Aooo『Rooom』アルバムのアートワークを基調にしている。ポップで遊び心のある、ドリッピング（絵の具垂らし）調の多色パレット。UIの主役はピンク→オレンジのグラデーション＋淡いブルーグレー背景。

呼称ポリシー（アプリ / サービス）

LiveLog はブラウザ／PWAベースだが、利用層（ライブ好き・ほぼスマホ）に最も伝わる「アプリ」を基本呼称とする。事業・対外文脈でのみ「サービス」を使う。「サイト」は受動的に聞こえるため避ける。

対ユーザー（LP・アプリ内・SNS・宣伝） →「アプリ」で統一
対ビジネス（事業説明・顧問・資料） →「サービス」
「サイト」 → 使わない（機能性が伝わらない）

※ PWA化でホーム画面常駐＝実体も「アプリ」。コピーは「ライブ参加の記録・管理アプリ」で統一。

マネタイズ戦略
月額モデルへ転換

買い切り → 月額サブスクへ大転換（2026-06）

門倉さん提案＋林合意。買い切りはサービス終了時にクレームが起きやすいため、月額ベースに変更。AI機能は独立した上位プランに分け、年/月の両方を用意。継続課金の燃料は「過去の記録」ではなく「未来のライブ発見（好きなアーティストの新公演ニュース）」に置く。

料金体系（3段階・各有料プランに年/月）

線引きの考え方：② 有料＝AIを使わないプレミアム（記録無制限・友達・UGCベースの未来通知＝原価ほぼゼロ）。③ AI機能＝LLMコストが乗るもの全部（Gmail取込・画像/メール解析・AI能動監視・プレイリスト）。
価格心理：②は月300×12=3,600 ＞ 年2,000／③は月600×12=7,200 ＞ 年6,000 ＝ 年額が明確にお得に見える（門倉案の狙い）。③は②に対し「+¥300/月・+¥4,000/年」とも見せられる。
月額成立の鍵：未来ライブニュース（継続課金の燃料）
課題：月額は「毎月価値を感じる理由」が要る。今のLiveLogは過去データ中心で、ライブに行かない月は価値ゼロに感じる＝解約されやすい。
解決：好きなアーティストのライブ情報をアプリが常に自動チェック（SNS・公式サイト・世の中すべて）し、ユーザーが「検討中」にする前段階で"ニュース"として先回り通知する。チケぴ等のアラートはしょぼく、多くの人はSNSで発見→見逃す。そこをLiveLogが代わりに網を張る。
効果：「行かない月も新公演ニュースが届く」＝毎月開く理由・払い続ける理由になる。価値が「過去（記録）→過去＋未来（発見）」に変わる。
情報源現実度プラン備考
UGC（誰かが登録→同アーティストのフォロワーに通知）◎ すぐ可能② 有料原価ゼロ・ユーザー増で強化（B-5）
Gmail（FC/メルマガのツアー告知）◎ 実装済連携で拾える③ AI本人会員のものに限る
AI Web検索（定期的にアーティスト名で検索）○ 精度・コスト要調整③ AI"世の中すべて"の近似
SNS自動監視（X/Instagram公式）△ API制限・高コスト③ AI（将来）完全自動は提携待ち
方針：完全自動の"全部監視"は現状ムリ。UGC（②）＋Gmail＋AI検索（③）の組み合わせで「主要アーティストは網羅、ニッチは徐々に」。未来ニュースの情報源で②/③を分ける（UGC=②／AI監視=③）のが自然。

① アー写連携（★1・土台）
