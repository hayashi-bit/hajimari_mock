## 1. サービスのコンセプト

**一言:** 「ライブを"点"じゃなく、前後ぜんぶの"体験"として楽しみ尽くす。同じ熱量の仲間とつながり、発信し合って、もっと興奮する。」

「記録(ログ)」は入口・手段であって本質ではない。本質は **体験 × つながり × 発信**。

### 提供価値の柱
1. **前後まで楽しむ** — 情報入手→検討→予習→当日へのカウントダウン→参戦→余韻→振り返り→カラオケ、という一連の"軸"がライブ
2. **見せ合うだけで分かり合える** — 音楽好き同士、リスト/参戦履歴を見せ合えば一瞬で通じ合える
3. **ワクワクを育てる** — 抽選→当選→当日とステータスが変わるたびに高揚（行く前のワクワクが特に重要）
4. **AIで楽に・楽しく** — 写真/メール/テキストから自動で取り込み、手入力ゼロを目指す
5. **発見と再燃** — 新しいアーティスト/ジャンルに出会い、行く回数が増える。友達も増える

### 主な機能
- ライブの記録・管理（予定/参加/不参加・13種ステータス・チケット期限管理）
- セットリスト記録、メモ・感想、友達のライブ履歴閲覧、公開プロフィール
- AI取り込み（画像/テキスト）、フェス出演者の自動補完
- クラウド保存でデータが消えない・マルチデバイス対応
- （構想）共通ライブの可視化、SNS映え画像、予習プレイリスト、フォロー中アーティストの新公演通知

### ターゲット
- コア: 10〜20代、現場が生活の一部のライブ愛好者（年100本級も）
- 拡張: 30〜40代の「久々に行こうかな」を刺激する層

## 2. ネーミングの条件（現在の希望）

- **ポップな響き**（若年層に刺さる・親しみ・かわいい/明るい）
- **「ライブ」という語を入れたい**
- **インテリジェンス/自動化のニュアンス**も出せると尚良し（"AI"そのものは難しくてもOK）
- 特に表現したいのは**「行く前のワクワク（そわそわ・高鳴り）」**（"余韻＝後ろだけ"は弱いと感じている）
- 被りを避けたい（後述）。理想は「**自区分に登録商標なし＋ドメイン/SNSハンドルが取れる**」
- 日本市場中心。読み（カタカナ）で覚えやすいこと

### 命名の前提（被りの考え方）
- 被り＝①法的リスク（**自分の区分で登録された商標だけ**が止める／区分は第9類アプリ・第41類エンタメ・第42類SaaS・第38類配信）、②実用リスク（検索・SNSで埋もれる）
- **「ライブ」は一般名詞で自由に使える**（識別力なし＝独占不可）。問題は組み合わせる固有語が既存と被ること
- 直球の「ライブ＋ノート/日記/ログ/ダイアリー」は**競合に全て先取り済**（LiveNote / LIVE DIARY / Livelog / LIVE x LIFE / Live Rock / OTOAKA 等）
- 「Live◯◯」「○○ライブ」は**ライブ配信アプリ**に密集（LiveMe / DokiDoki Live / LIVEE! など）→ 配信系との混同に注意

## 4. 現時点の結論・他AIに聞きたいこと

- **総合力の一推し: ソワライブ（sowalive）**。被り○・.com取得可・ポップ・"行く前のワクワク"を表現。
- **自動/賢いを効かせるなら: スマライブ（smalive）**。
- まだ「これだ！」と確信に至っていないので、**新しい角度の候補**がほしい。
