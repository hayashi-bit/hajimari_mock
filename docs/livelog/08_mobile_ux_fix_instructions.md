# Manus向け 修正指示書：モバイルUX（M-1〜M-6 まとめて修正）

対象: LiveLog本体（Manus）
元フィードバック: `07_mobile_ux_feedback.md`（テスター：加藤ゆかり・2026-06-17）
検証用: `09_mobile_ux_verification_checklist.md`

以下6件をまとめて修正してください。**P1（M-1〜M-3）はモニター配布前の必須**、P2（M-4〜M-6）は分かりやすさ向上。

---

## M-1【P1】トーストが下のタブバー/ボタンと被って読みにくい

- 症状: 画像取込後などのトースト（例「ライブ情報を読み取りました！内容を確認して取り込んでください」「更新しました」）が、画面下部の**タブバー（これからのライブ/ライブの記録/友達/設定）と重なり**、文字とアイコンが被って読めない。**おそらく全トースト共通**。
- 修正:
  - トーストの表示位置を、**タブバーの高さ＋余白の分だけ上げる**（`bottom` をタブバー高さ＋16px 以上に）。
  - モバイルのセーフエリア（`env(safe-area-inset-bottom)`）も考慮。
  - 共通トーストコンポーネントを1か所直せば全画面に効くようにする。
- 完了条件: どの画面のトーストもタブバーに重ならず、全文が読める。

## M-2【P1】ライブ削除の確認が素のブラウザconfirm

- 症状: 削除時に `window.confirm` の「**livelogapp-…manus.space の内容／このライブを削除しますか？**」が出る。ドメイン名が露出し不格好。
- 修正:
  - `window.confirm` を**アプリ内の確認モーダル**に置換。
  - 文言例:「このライブを削除しますか？」＋**対象の公演名**を表示／ボタン「キャンセル」「削除」（削除は赤）。
- 完了条件: 削除時にドメイン名の出ないアプリ内モーダルが出る。

## M-3【P1】「null」がそのまま表示される

- 症状: 取込候補やライブ詳細で「**発券（null）**」「**日本武道館（null）**」のように `null` が文字表示される。
- 修正:
  - 値が `null` / 空文字 / `"null"` 文字列のとき、**その項目を表示しない**、または「—」「未設定」に置換。
  - 会場の都道府県など `(null)` の括弧書きも、値が無ければ括弧ごと出さない。
  - 表示用の共通ヘルパ（例 `displayOrDash(v)`）を作り、該当箇所に適用。
- 完了条件: 画面のどこにも "null" が出ない。

## M-4【P2】入力欄に何を書くか分からない

- 症状: 入力欄（受付開始日時など）が空で、何を入れる所か不明。
- 修正: 各入力欄に**グレーのプレースホルダ例**を入れる（下の「チケットサイト」URL欄＝`https://eplus.jp/... など` のイメージ）。
  - 例: 受付開始 → `2025-10-01`／時刻 → `10:00`／会場 → `渋谷FOWS` など。
- 完了条件: 主要入力欄にプレースホルダ例が表示される。

## M-5【P2】「画像から取り込む」のガイドが無い

- 症状: どこからどこまでの情報があれば取り込めるか分からない。
- 修正:
  - 「画像から取り込む」近くに**ヘルプ（別ページ or モーダル）**を用意。
  - **OK例**（公演名・日付・時刻・会場が写っている告知画像）と **✕例**（日付が入っていない等）を提示。
  - 「読み取れる項目：公演名 / 日付 / 時刻 / 会場 / 出演者」を明記。
- 完了条件: 画像取込の前に「何が必要か」が分かるガイドがある。

## M-6【P2】取込可能な画像枚数の明記

- 症状: 何枚まで取り込めるか不明。1枚だと情報が入り切らない場合がある。
- 修正:
  - 取込可能枚数を**明記**（例「最大◯枚まで」）。
  - 可能なら**複数枚アップロード対応**（フェスのTT画像など複数枚を1公演にまとめる）。
- 完了条件: 枚数上限が画面に表示される（複数枚対応は可能なら）。
