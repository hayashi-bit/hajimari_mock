# モバイルUXフィードバック（要修正予定）

記録日: 2026-06-17
出所: 実機モバイル検証（テスター：加藤ゆかり）／スクショはセッションに添付
ステータス: 要修正（未着手）

LiveLog本体（Manus）側の改善項目。優先度は P1=体験を明確に損ねる / P2=分かりやすさ向上。

| # | 区分 | 内容 | 期待する修正 | 優先 |
|---|---|---|---|---|
| M-1 | トースト被り | 画像取込後などの**ポップアップ（トースト）が下のタブバー/ボタンと重なって読みにくい**。おそらく**全トースト共通**でボタンに被っている。 | トーストの表示位置を上げる／タブバーの上にマージンを確保。全画面のトーストに適用。 | **P1** |
| M-2 | 削除ダイアログ | ライブ削除時に「**livelogapp-…manus.space の内容／このライブを削除しますか？**」という素のブラウザconfirmが出る。ドメイン名が出て不格好。 | アプリ内の確認モーダルに置換（「このライブを削除しますか？」＋公演名表示、キャンセル/削除）。 | **P1** |
| M-3 | null表記 | 取込結果に「**発券（null）**」「**日本武道館（null）**」のように **null がそのまま表示**される。 | 値が無い項目は **null を表示しない**（非表示）か「—」「未設定」等に置換。 | **P1** |
| M-4 | 入力欄の例示 | 入力欄が**何を書く所か分からない**。 | 各入力欄に**グレーのプレースホルダ例**を入れる（下のチケットサイトURL欄のイメージ＝`https://eplus.jp/...` など）。 | P2 |
| M-5 | 画像取込のガイド | 「画像から取り込む」で、**どこからどこまでの情報が必要か分からない**。 | **別ページで例を提示**（OKの例／✕の例＝「日にちが入っていない」等）。良い告知画像/悪い画像のサンプル。 | P2 |
| M-6 | 画像枚数の明記 | **何枚まで取り込めるか**の記載が無い。1枚だと情報が入り切らない場合がある。 | 取込可能枚数を明記（例「最大◯枚」）。複数枚対応の検討。 | P2 |

## 補足
- M-1・M-2 は「見た目の完成度」に直結するので、モニター配布前に直したい（第一印象に影響）。
- M-3 の null は LLM抽出で空値のときの表示制御。フロント側で `null/空` を出さない処理を入れる。
- M-5/M-6 は「画像から取り込む」機能のオンボーディング。別ページのヘルプ or ツールチップで対応。
