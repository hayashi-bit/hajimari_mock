# メール取込フィードバックシート（漏れ・誤り・チューニング用）

> Gmailからのライブ自動取り込みを良くするための記録用です。
> **送る情報は「送信元アドレス」と「件名」だけ**で大丈夫です（本文は不要・こちらも本文は保存しません）。
> これが集まるほど、マイナーなライブまで取りこぼさなくなります。

## 使い方
- ライブ詳細やメールを見ながら、当てはまる表に1行ずつ書いてください。
- 分かる範囲でOK。空欄があっても大丈夫です。

---

## ① 漏れ（行ったのに取り込まれていないライブ）
| ライブ名 / 公演名 | 日付 | 会場 | チケットを買った所（ぴあ/eplus/ローチケ/直販 等） | メールの送信元アドレス | メールの件名 |
|---|---|---|---|---|---|
| 例）前髪ぱっつん少年 ぱっつん化計画 | 2025-03-31 | Flowers Loft | チケットぴあ | tickets@pia.jp | 【チケットぴあ】お申し込み完了 |
|  |  |  |  |  |  |
|  |  |  |  |  |  |

## ② 誤り（ライブじゃないのに入ってしまったもの）
| 入ってしまった内容 | これは何？（野球/広告/通販/メルマガ 等） | メールの送信元アドレス | メールの件名 |
|---|---|---|---|
| 例）◯◯戦 チケット | 野球 | info@baseball.example | 【完了】観戦チケット |
|  |  |  |  |
|  |  |  |  |

## ③ 情報の間違い（取り込めたが内容がズレている）
| ライブ名 | 間違っている項目（日付/会場/アーティスト/形式） | 正しい内容 |
|---|---|---|
| 例）◯◯ワンマン | 会場が空欄 | Zepp Haneda |
|  |  |  |
|  |  |  |

## ④ フライヤー・出演者・アー写の間違い
| ライブ名 | どこが変？（フライヤー誤画像/出演者違う/アー写出ない） | 補足（分かれば） |
|---|---|---|
| 例）◯◯フェス | フライヤーが別公演の画像 | 差し替えボタンは効いた |
|  |  |  |
|  |  |  |

---

## チューニングの観点（運営メモ・モニターは無視でOK）
- ①漏れの送信元ドメインを、取り込み対象に追加（広い網を維持）。
- ②誤りの送信元/件名パターンを、安く間引く事前フィルタ or Step2分類のヒントに。
- ③④は抽出プロンプト（公演名/日付/会場/出演者/ライブ形式）の精度改善材料。
- 個人情報の最小化：収集は**送信元＋件名のメタデータのみ**。本文は見ない/保存しない。
