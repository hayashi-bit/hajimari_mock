## B. 新機能：アーティスト名・ライブ形式の自動登録

### 目的
公演を登録/取込した時に、**アーティスト名**と**ライブ形式**を自動で埋める。手入力の手間を消し、
「履歴が勝手に整う」体験にする（コア価値＝過去の自動復元と地続き）。

### スキーマ（無ければdev/prod両方に追加）
- `lives.artistName VARCHAR(255) NULL`（主アーティスト／出演者）
- `lives.liveFormat VARCHAR(32) NULL` — 値の語彙：`oneman`(ワンマン) / `taiban`(対バン) / `festival`(フェス) / `event`(イベント) / `unknown`
- ※マイグレーションはdev→確認→prodの順。webdev_execute_sqlは本番DB固定に注意。

### 抽出ロジック（Gmail取込のStep3拡張＝メタデータのみ・本文非保存）
既存のextractEmailData（公演名/日付/会場を抽出）に項目を追加：
```
"artistName": "主アーティスト名（出演者）。複数なら主たる1組。不明なら null",
"liveFormat": "oneman|taiban|festival|event|unknown",
"artistConfidence": 0.0〜1.0
```
**判定ヒント（プロンプトに入れる）：**
- 公演名に「ワンマン」「ONE MAN」→ oneman
- 「対バン」「vs」「×」複数アーティスト併記 → taiban
- 「FES」「フェス」「FESTIVAL」「〇〇祭」 → festival
- イベント名のみで出演者が読み取れない → event / artistNameはnull可
- 確信が持てなければ unknown / null（誤登録より空を優先）

### 手動登録時の補完
- 手動で公演名を入れた時も、同じLLMで artistName / liveFormat を**候補サジェスト**（確定はユーザー）。
- 既に値が入っている公演は上書きしない（冪等）。

### UI
- カードにアーティスト名を主役表示、ライブ形式はバッジ（ワンマン/対バン/フェス）。
- 自動推定値は「自動」マーク＋ワンタップ修正可。誤りはユーザーが直せる。

### 既存データのバックフィル（任意・非破壊）
- 既存livesで artistName/liveFormat が空のものに対し、公演名から推定して**候補として**埋める
  （いきなり確定せず、まとめて確認できると親切）。本番データは上書きしない方針で。

### Acceptance
- Gmail取込時に artistName / liveFormat が自動で入る（不明はnull/unknownで誤登録しない）。
- ワンマン/対バン/フェスが公演名から妥当に分類される（数例で検証報告）。
- 既存値を上書きしない。ユーザーが修正できる。

### 非交渉
- 本文非保存（抽出はメタデータのみ）。プレビュー→確認を維持。
- スキーマ変更はdev/prod両方。テスト通過後チェックポイント。公開は林確認後。
- 実装後 changelog 更新。
