# 友達登録機能＋友達バッジの復活（Manus指示・最優先バグ）

作成: 2026-06-27 ／ 種類：後退バグ（昨日まで使えた友達追加・友達バッジが消えた）。Manus（現本番）で今すぐ。
関連: doc52(A)／doc41(公開設定・相手プロフィールのカード全体タップ・バッジ5-1)。
※友達追加とバッジは同じ友達まわりの変更で同時に消えた可能性が高い＝原因は1つかも。

## マナスへの指示（全文）

```
【最優先・後退バグ修正：友達登録機能＋友達バッジの復活】
症状：①プロフィール画面は表示されるが「友達を追加する」導線が消えている
②友達との「バッジ」機能も消えている。昨日までは両方使えた＝最近の変更で消えた後退バグ。
友達追加とバッジは同じ友達まわりの変更で同時に消えた可能性が高い。本番データは壊さない。

■1. 原因調査（まず直さず原因を特定）
- 友達追加のUI（「友達を追加する」ボタン、相手プロフィールURL(/u/:publicId)からの追加、
  共有からの追加 等）と、友達バッジの表示が、最近のどの変更で削除/非表示になったかを特定する。
- 確認場所の例：友達ページ、プロフィール表示コンポーネント、バッジ表示コンポーネント、
  profileRouter の友達追加・バッジ算出ロジック、表示条件（公開設定/ログイン状態の分岐で隠れてないか）。
- 「UIだけ消えたのか」「API/ロジックも消えたのか」を切り分けて報告。両者が同一原因かも報告。

■2. 復活
(友達追加)
- 友達追加のフローを戻す：相手のプロフィールURL(/u/:publicId)や共有/検索から友達を追加できる。
- friendships（友達関係）への追加が動く。重複追加・自分自身の追加を防ぐ（冪等）。
- 追加後、相手のライブ記録が並ぶ（公開範囲に従う）。
(友達バッジ)
- 消えた友達バッジを昨日までの状態に戻す。バッジは「友達との参戦の重なり方」を表す3種：
  ① 同じライブに行ってた（同一公演に両者が参加＝記録が一致）
  ② 一緒に参戦（同じ公演に“一緒に”行った＝同行）
  ③ 同じフェスの違う日参戦（同一フェスの別日にそれぞれ参加）
  ※言葉(ラベル)は以前検討した表現を踏襲。最終文言は林確認で微調整可。
- バッジの算出（友達との共通公演・同行・同フェス別日）が正しく出ること。
- doc41整合：相手プロフィールはカード全体タップ可／公開設定(全公開/フォロワーのみ)と矛盾しない。

■3. 安全
- 本番DBへの破壊的変更(DELETE/DROP等)はしない。検証はdev serverで。
- 既存の友達データ(friendships)を壊さない。

■完了報告
- 消えた原因（どの変更で・友達追加とバッジは同一原因か・UIだけかロジックもか）。
- 復活した内容（友達追加の導線／バッジの表示）。
- Acceptance：実際に友達を追加でき相手のライブ記録が並ぶ／友達バッジが表示される。
```
