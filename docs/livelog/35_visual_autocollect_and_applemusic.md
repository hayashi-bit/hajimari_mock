## D-1. フライヤー画像（AI自動収集・候補提示型）

### パイプライン（doc27の公式サイト検索と同型）
1. **検索クエリ生成**：`{artistName} {eventName} {date} フライヤー` ／ 取れなければ `{artistName} {venue} ライブ フライヤー`。
2. **実Web検索（Tavily）**：画像URL候補を実在の検索結果から取得（URLを生成・捏造しない）。
3. **LLMで候補選別**（下のプロンプト）：検索結果（タイトル＋URL＋スニペット）から、その公演のフライヤーとして最も妥当な画像URLを1つ選ぶ。該当なしなら `null`。
4. **生存チェック**：選ばれたURLにHEAD/GETして 200＋Content-Typeがimage/*か確認。404・非画像なら不採用。
5. **候補としてプレビュー表示**：`flyerImageUrl` には**即書き込まず**、まずユーザーに「候補です・採用/差し替え/削除」で見せる。採用で初めて保存。
6. 採用後の公開範囲は friends 初期値。

### LLM選別プロンプト（system）
```
あなたはライブイベントのフライヤー画像を、検索結果から1つだけ選ぶ係です。
入力は「アーティスト名・公演名・日付・会場」と、Web検索結果のリスト
（各itemは title / url / snippet / image_url）。
ルール：
- 検索結果に実在するimage_urlの中からのみ選ぶ。URLを自分で創作・改変しない。
- その公演（アーティスト/公演名/日付/会場の一致度が高い）のフライヤー/ポスター画像を優先。
- アー写単体・ロゴ・無関係画像・チケット転売サイトのサムネは避ける。
- 確信が持てない場合は無理に選ばず null。
出力は純粋なJSONのみ。コードブロックや説明文を付けない。
形式：{"image_url": "https://..." または null, "reason": "短い理由"}
```
※パースは parseLLMJson（de-fence＋ブレース切り出し）を使う。

### Acceptance（D-1）
- 小規模アーティストでも「無関係画像を確定添付しない」（誤りは候補段階で弾ける/ユーザーが消せる）。
- 採用するまで flyerImageUrl は空のまま。採用後 friends 初期値で表示。
- 前髪ぱっつん少年など実リストの数組で候補が出るか・誤爆しないかを報告。

## E. Apple Music アー写カバー率検証 → 良ければアー写＋予習PL

### 認証（env・.p8は秘密）
- `APPLE_TEAM_ID=6ZZDG46JS2`
- `APPLE_KEY_ID=867859YX46`
- `APPLE_PRIVATE_KEY`（.p8の中身）＝**env経由のみ。チャット・コード・リポジトリに絶対貼らない。**
- TEAM_ID/KEY_IDもコードに直書きせずenvから読む。

### 手順
1. `tools/apple_music_eval/run_eval.py` を上記env3点で実行（JWT ES256・kid=KEY_ID・iss=TEAM_ID・exp≤12h）。
   - `GET https://api.music.apple.com/v1/catalog/jp/search?types=artists&term=<artist>`
   - `data.results.artists.data[].attributes.artwork.url` の有無で「アー写OK」判定。
2. `fixtures.py` の実参戦リスト16組（前髪ぱっつん少年・meiyo・ロクデナシ・ずっと真夜中でいいのに。・Aooo・紫今・アザミ・ポップしなないで・RuLu・NOMELON NOLEMON・水槽・はしメロ・CLAN QUEEN・長瀬有花・MAISONdes・チョーキューメイ）で「アー写OK n/16」を集計。
3. **最重視＝前髪ぱっつん少年が取れるか**を明記（ルール3：コアユーザーが見るマイナーが取れてこそ価値）。

### 結果が良ければ実装（カバー率の表を添えてから）
- **アー写表示**：アーティストにApple Musicのartwork.urlを紐付けて表示（artwork.urlは`{w}x{h}`を実寸に置換）。
- **予習プレイリスト**：出演者の代表曲（Apple Musicの top songs / catalog）を並べ、公演前の「予習」導線にする（コンセプト＝発見・予習）。

### Acceptance（E）
- 「アー写OK n/16」の比較表を提出（ルール3の実リスト検証の表）。
- 前髪ぱっつん少年の可否を明記。
- 採用判断（n/16が実用水準か）を添える。良ければアー写＋予習PLへ。
