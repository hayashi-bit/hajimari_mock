# Apple Developer：個人登録の方針＋法人切替の閾値

作成: 2026-06-18

## 現状の方針：個人（Individual）で登録
理由（2026-06-18判断）：
- 現状は**ブラウザアプリ**＝App Store掲載が無く、Apple Developerの**販売者名（会社名）は表に出ない＝法人のブランド効果ゼロ**。
- **Apple Music API / MusicKit の性能・機能は個人も法人も完全に同じ**（アー写取得に支障なし）。
- 法人は D-U-N-S番号・会社サイト・法的権限が必要で**承認に数週間**かかる＝今は不要なコスト。
- アプリ化（ネイティブ公開）は「売れたら」＝未来・不確定なので、そのとき法人を立てればよい。

→ **今は個人で登録し、Apple Music APIを最速で動かす。**

---

## 法人（Organization・スリーウェイ）へ切り替える閾値
**下記のどれかに達したら、クロコが法人化を提案する。**

1. **ネイティブアプリ（iOS）をApp Storeに公開すると決めたとき**（最有力トリガー）
   - 会社名義の販売者名で出すなら法人アカウントが必要。
2. **会社名義で販売者名を出し、ブランド/信頼を確立したいとき**。
3. **事業として軌道に乗ったとき**（目安：**有料100人達成／MRRが継続的に立つ／7月ゴール達成後**）。
4. **アカウントを会社所有・チームで管理したくなったとき**（担当者が増える・引き継ぎが要る）。
5. **投資・提携・事業売却など、対外的に法人実体が必要になったとき**。

### 前倒し準備のメモ
- **D-U-N-S番号の取得は数週間かかる**ため、上記が「見えてきた」段階で、**D-U-N-S申請だけ先行**しておくと法人化がスムーズ。
- 法人化時のAccount Holderは**法的署名権限を持つ人**（林さんに権限があればOK）。
- 個人で公開済みアプリがある場合、法人へは**アプリ移管**の手続きが要る（ゼロからではない）。

### クロコのアクション
- 上記閾値の兆候（特に「ネイティブ公開を検討」「有料100人達成」）を察知したら、**こちらから法人化を提案する**。
